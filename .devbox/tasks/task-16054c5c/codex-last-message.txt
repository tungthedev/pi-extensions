This repo is a Bun/TypeScript package of Pi coding-agent extensions. The package entrypoint is the `pi.extensions` array in [package.json](/Volumes/Data/Projects/exp/pi-extensions/package.json), and the top-level overview is in [README.md](/Volumes/Data/Projects/exp/pi-extensions/README.md). In practice, Pi loads each `extensions/*/index.ts` file and each one registers UI hooks, tools, commands, or prompt/session behavior.

The repo breaks down like this:

- `extensions/editor`: replaces Pi’s default editor UI with a Codex-style boxed composer and status row. See [extensions/editor/index.ts](/Volumes/Data/Projects/exp/pi-extensions/extensions/editor/index.ts).
- `extensions/mermaid`: detects Mermaid blocks in chat, renders them inline, and exposes a viewer command/shortcut. See [extensions/mermaid/index.ts](/Volumes/Data/Projects/exp/pi-extensions/extensions/mermaid/index.ts).
- `extensions/web-search`: adds Gemini-backed `web_search` and `web_extract` tools. See [extensions/web-search/index.ts](/Volumes/Data/Projects/exp/pi-extensions/extensions/web-search/index.ts).
- `extensions/cloudflare-crawl`: adds a `crawl_page` tool backed by Cloudflare Browser Rendering. See [extensions/cloudflare-crawl/index.ts](/Volumes/Data/Projects/exp/pi-extensions/extensions/cloudflare-crawl/index.ts).
- `extensions/codex-content`: the biggest compatibility layer. It disables some built-in Pi tools and replaces them with Codex-style file/shell/apply-patch/workflow tools plus TUI renderers. See [extensions/codex-content/index.ts](/Volumes/Data/Projects/exp/pi-extensions/extensions/codex-content/index.ts) and [extensions/codex-content/compatibility-tools/index.ts](/Volumes/Data/Projects/exp/pi-extensions/extensions/codex-content/compatibility-tools/index.ts).
- `extensions/codex-system-prompt`: injects Codex-flavored system prompt text before agent start, including personality/profile handling. See [extensions/codex-system-prompt/index.ts](/Volumes/Data/Projects/exp/pi-extensions/extensions/codex-system-prompt/index.ts).
- `extensions/codex-subagents`: implements Codex-style subagent orchestration like spawn/send/wait/resume plus persistence and profile resolution. See [extensions/codex-subagents/index.ts](/Volumes/Data/Projects/exp/pi-extensions/extensions/codex-subagents/index.ts) and [extensions/codex-subagents/subagents/index.ts](/Volumes/Data/Projects/exp/pi-extensions/extensions/codex-subagents/subagents/index.ts).
- `extensions/ext-manager`: in-app extension manager UI for discovering and toggling local/package extensions via `/extmgr`. See [extensions/ext-manager/index.ts](/Volumes/Data/Projects/exp/pi-extensions/extensions/ext-manager/index.ts) and [extensions/ext-manager/controller/index.ts](/Volumes/Data/Projects/exp/pi-extensions/extensions/ext-manager/controller/index.ts).
- `extensions/skill`: skill discovery/loading glue for Pi’s skill system. See [extensions/skill/index.ts](/Volumes/Data/Projects/exp/pi-extensions/extensions/skill/index.ts).

The rest is support structure: `docs/plans` holds design/implementation notes, tests sit next to extensions, and the toolchain is intentionally small: Bun, TypeScript, Oxlint, Oxfmt. If you want, I can also give you a “how a Pi session flows through these extensions” walkthrough next.
